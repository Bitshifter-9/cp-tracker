# Environment Variables

## Development (.env files - DO NOT COMMIT)

### Backend (.env)
```
MONGODB_URI=mongodb+srv://your-username:your-password@cluster.mongodb.net/cp-tracker
JWT_SECRET=your-super-secret-jwt-key-change-this-in-production
PORT=5001
```

### Frontend (.env)
```
VITE_API_URL=http://localhost:5001/api
```

## Production (Render.com)

### Backend Environment Variables
Set these in Render Dashboard → Your Backend Service → Environment:

| Variable | Value | Notes |
|----------|-------|-------|
| `MONGODB_URI` | `mongodb+srv://...` | Your MongoDB Atlas connection string |
| `JWT_SECRET` | Auto-generated by Render | Or set your own secure random string |
| `PORT` | `5001` | Already configured in render.yaml |
| `NODE_ENV` | `production` | Already configured in render.yaml |

### Frontend Environment Variables
Set these in Render Dashboard → Your Frontend Service → Environment:

| Variable | Value | Notes |
|----------|-------|-------|
| `VITE_API_URL` | `https://your-backend.onrender.com/api` | Replace with your actual backend URL |

## Getting Your MongoDB URI

1. Go to [MongoDB Atlas](https://cloud.mongodb.com)
2. Click "Connect" on your cluster
3. Choose "Connect your application"
4. Copy the connection string
5. Replace `<password>` with your actual password
6. Replace `<dbname>` with `cp-tracker`

Example:
```
mongodb+srv://myuser:mypassword@cluster0.abc123.mongodb.net/cp-tracker
```

## Security Notes

- ⚠️ **NEVER commit .env files to Git**
- ⚠️ **NEVER share your JWT_SECRET or MongoDB password**
- ✅ `.env` files are already in `.gitignore`
- ✅ Use strong, random passwords
- ✅ Rotate secrets if compromised
